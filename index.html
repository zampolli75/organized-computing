If you are in the college of business, you know that Information Technology has, over the last thirty years, revolutionized every aspect of business, from Accounting to Strategy.

What he meant is that just like a person on a bicycle can go further and go faster than a person walking, a computer can help your mind work faster and go further than you would achieve alone. As a knowledge worker you will be doing most of your work with your mind.

Your goal in this course is to take advantage, rather than being hurt, by the continuing shift toward the digitization of work. If you successfully complete the course you will become what we call a sophisticated end-user. In four months you will be a savvy knowledge worker by developing expertise in three distinct but related domains: Practical computing skills, foundational information technology concepts, and foundational information systems concepts. This class is your chance to learn to ride the "bicycle of the mind" effectively and efficiently.

Basic Concepts

There are a number of basic concepts that represent the required vocabulary of any sophisticated end-user. They are briefly introduced below and you should master them, if you have not already.

  • File: In a computer systems a file is a data resource, available to system users, that is capable of being manipulated as an entity. When you delete an embarassing Instagram picture from your iPad, you are acting as a system user who is manipulating a file. But system users include the system itself (e.g., the operating system of your smartphone) and its application programs. The type of manipulations performed on a file include moving, copying, deleting, etc.

    There are two major classes of files: data files and program files. A data file, such as that embarassing Instagram picture you were trying to delete earlier, is a file that holds data to be used by an application. A data file does not instruct the computer on how to perform a task, but simply serves as a repository of “information.” Conversely, a program file contains the instructions, or steps, the computer needs to perform a tasks. Microsoft Excel is a program file.

  • Metadata: Data that provides information about other data is called metadata. Thus, the role of metadata is to be a descriptor for other data. You are certainly familiar with file extensions like .docx (for Microsoft Word files) or .xlsx (for Microsoft Excel files). File extensions are a good example of metadata. They serve the purpose of describing the type of file to the operating system so that the operating system can associate the correct application to it. In personal computers, extensions are usually three letters, but this is not a requirement.
  • Directory: A file system, like the one on your personal computer, must be able to reference all the files it contains. How else could you manipulate them after all? Directories, also called folders, are those structures in your file system which contain references to other computer files. Directories (or folders) can be logically nested, in which case they are called subdirectories (or subfolders). Finally, the term "working directory" refers to the currently active directory, the one in which the system user is currently operating.
  • Volume (or logical drive): A single accessible storage area with a single file system. A typical computer system can have both internal and external volumes. The best example of an internal volume is the solid state drive (SSD) of your smartphone. It is the storage area that your smartphone uses to save the operating system, applications and data. However, many of you also use external volumes for backup or simply becacuse you have so much stuff on your phone that you run out of space on your internal volume. Apple iCloud or Google Drive are examples of external volumes.
  • File Path: A string of characters, or other symbolic representation, that specifies the exact location of a file in a volume. While this definition sounds intimidating, you use file paths all the time. The last time you had to save a file that a friend shared with you, you had to specify the file path for the file system to know how to reference the location to store and later retreive the file. An absolute file path specifies the location of the file starting from the volume. A relative file path specifies the location of the file relative to the current working directory.
  • Alias (or shortcut): Sometimes it is useful to create small files that represents, or point to, objects in a volume. These are called alias in Mac OS and shortcuts in Microsoft Windows. The objects that the alias refers to are any objects in the file system, like a file, directory, subdirectory, etc..
  • File structure: A file structure captures the way your files are organized in your computer system. The two major types are the hierarchical and tag structures. The hierarchical structure organizes files using the metaphore of a cabinet. A cabinet has drawers that contain folders which in turn contain individual files. If a file is in one folder, it cannot be in another folder at the same time. This is the most common file structure. A tag structure organizes files by labeling them. Thus, a file can have multiple labels at the same time.

Basic Skills

You have been using computers for the majority of your life. Therefore, there are a number of important computer skills that we assume you have already mastered. We list them below, if you don’t feel comfortable with any of these make sure that you acquire them immediately.

  • You have some prior knowledge with Microsoft Office applications in general and with Word and Excel specifically. While our learning material starts from the very basics of each applications, we assume you already know how to create a basic Word Document and Excel Worksheet. We also take it as given that you can quickly self-learn any of the basic skills that you may be missing.
  • You are able to use your prior knowledge of computing to navigate the small potential differences that can occur between available instructions and the specifics of your application. Software programs are deceptively complex. They have thousands of features and functionalities. Your specific application may have some special configurations or customization. We assume that these minor inconsistencies will not stop you cold in your tracks. We take it as given that you can either navigate these differences independently or seek appropriate help to correct them.
  • You have the basic file management skills of a functional computer user. Specifically, you know how to organize your files in meaningful storage hierarchies that make it easy to find them, and you know how to use conventional and intuitive naming conversions for them. You understand the role of file extensions, and you can manage them appropriately.
  • You understand the importance of saving your work and backing up your files. Specifically, we take it as given that during a working session (e.g., completing your homework) you remember to save after completing any substantive task. This is because you understand that primary memory is volatile and that computers can, and do, freeze from time to time. Thus, were the computer to lose power for any reason, your unsaved work would be lost. We also take it as given that you appreciate the importance of backups, whether on peripherals (e.g., a USB drive) or remotely (e.g., on a cloud service like Dropbox). This is because you understand that secondary storage devices are subject to equipment failure. Albeit rare, when equipment failure happens it can be very costly. Thus, the appropriate risk management strategy is to develop sound backup practices. In the case of this course you should have at least two separate copies of each important file at all times.
  • You have basic time management skills and you know how to successfully organize your work. One day you will be a manager, which means you will be responsible for organizing the work and efforts of your employees, in some cases, hundreds or even thousands of them. At this stage in your professional life you are very lucky, all you have to do is organize your own self. We take it as given that you can do so. If you find yourself unable to do so reach out to us immediately.
  • If any of the above assumptions is not correct in your specific case, or if any of the terms and concepts we refer to in the above section is foreign to you, it is your responsibility to contact a teaching assistant as early as possible and remedy your current skill deficiency.

Our Pedagogical Approach: The Flipped Classroom

Practical computing skills are widely distributed amongst the population of students in this class. Some of you have considerable experience with Microsoft Office and may have taken formal courses in high school. Sometimes these courses are fairly advanced. Many of you have limited knowledge of Excel, knowledge that comes from independently working in the application and learning from friends.

With such a variety of skills in the course the flipped classroom is the optimal approach to ensure a high degree of sophistication of all participants by the end of the course, while maintaining peak efficiency for each of you. The flipped classroom concept prescribes that high-quality self-directed learning modules be made available to the learners, along with target learning outcomes and a schedule of achievement expectations.

Armed with these three tools each of you can organize the learning schedule as needed, depending on your prior knowledge. If you are already very advanced you could skim the materials to make sure that you do know it and that you are aware of the many different approaches available in the application to perform the same task. Doing so enables you to verify that you have not picked up any bad habits and inefficient approaches in your previous experiences. If you are new to the application or the particular set of skills required to perform the week’s tasks, you should allocate significant time to review and practice the available material. Some words of caution and suggestions:

  • For as advanced as you may be, chances are you can learn quite a bit from the material in this course. There are often many ways to perform the same task in software applications. Reviewing the materials may show you new approaches that make you even more effective and efficient.
  • As mentioned above, you may be able to perform the task while being completely unaware that, because of vicarious learning, you picked up your friends’ bad habits. Thus, while you could complete the homework without reviewing the material, you may be nowhere near as efficient as you should be. This inefficiency will be very costly during the exam, where you are required to complete many tasks under time constraint. More importantly, these bad habits will hold you back in your career, where efficiency is a premium skill of the modern knowledge worker.

The ability to organize your learning is a great advantage also for those of you that come to the course without a strong background in Microsoft Excel. The material is available to you 24 hours a day, 7 days a week. You can repeat a unit as many times as you need, and you can return to the material while you complete the homework.

Of course, while we have high quality self-directed learning material available to you, we are here to enable your learning. But as you can see from this design of our pedagogy, there is one big assumption behind our approach: That you are a mature adult, conscious of the investment of time and money you are making in your education. We count on you to realize that you are doing homework for yourself, not for us. We are eager and happy to help you, but you are in charge and responsible for your own success.

Naming Convetions and Backups

When organizing your files it is a good habit to have a standard naming convention. That is, you should get used to naming files according to a set of rules that you apply consistently. Doing so will save you time and reduce the chance of you losing valauble assets. When creating a naming convention, keep in mind these suggestions:

  • Be descriptive but succinct
  • Be consistent in applying the naming convention
  • Avoid special characters (e.g., %, &, @)
  • When creating versions of your file use a consistent versioning approach such as appending a date (e.g., filename_20170824), or a version number (e.g., filename_v2.3).

Modern computer systems have a lot of storage space. For this reason you should maintain multiple copies of your files by creating versions as needed. When you do so:

  • Be descriptive but succinct
  • Be consistent in applying the naming convention
  • Avoid special characters (e.g., %, &, @)

We advise you to back up your files to protect your work from unnecessary loss. We suggest two main options:

  1. External Hard Drives: A flash drive or a hard disk offers the simplest option to create an entire back up of your computer. Both Windows and Mac users have system default programs that will allow you to create backups easily. Furthermore, you might want to schedule automatic backups so you can free your mind from having to remember to do it periodically.
  2. Cloud: Increasingly, there are free and commercial services to store your files and backups “in the cloud.” We will discuss “the cloud” in class during the course. For now, think of it as a flash drive located remotely and accessible via the Internet. Services such as Microsoft One Drive, Google Drive, Dropbox, or Apple iCloud are amongst the most popular. Installing the desktop version of these cloud services will provide you with automatic sync and backup of your files.

When thinking about your backup strategy, keep in mind these suggestions:

  • Isolate your data from system files
  • Design a storage system that is easy to move
  • Use a continuous backup system, not a manual system
  • Use a remote backup (e.g., cloud) if possible
  • Add a local backup using an external volume
